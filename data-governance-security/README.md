# Data Governance & Security — Learning Series

> DE-2026 · Depth-first · **Grounded in 2026 DE research** (Jul 2026) · **Tier 3 (senior differentiator)**
> Make data **trustworthy, compliant, and secure**. As pipelines feed dashboards and LLMs, governance is what keeps the business (and the regulator) happy.

## 🎯 What 2026 roles demand (research-grounded)

- **Access control** — authentication vs authorization, **RBAC**, row/column-level security.
- **Encryption** at rest & in transit; **secrets management** (vaults / key management).
- **PII & privacy** — masking, tokenization, anonymization.
- **Governance** — data **catalogs** (Purview, DataHub, Unity Catalog), **lineage**, quality/observability.
- **Compliance** — GDPR, HIPAA, SOC 2, and **India's DPDP Act**.

**Why Tier 3:** juniors rarely think about it; seniors design for it from day one. Increasingly critical in the **AI era** (bad/exposed data → breaches + LLM leakage).

**Practice:** apply on cloud (Azure RBAC, Key Vault, Purview) + the OrderIQ pipeline.
