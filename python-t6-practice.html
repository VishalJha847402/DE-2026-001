Design Transaction context manager with rollback
🎯 Scenario: Common senior DE interview. Build a context manager that commits on success and rolls back on error.
Task: Transaction class. __enter__ prints "BEGIN". __exit__: if exception (exc_type is not None) print "ROLLBACK", else print "COMMIT". Return False so exceptions propagate.
HintCheck if exc_type is None: — if so, COMMIT. Else ROLLBACK.
🐍 transaction.py
loading runtime…
Output will appear here…

✨ Solution

Solution
class Transaction: def __enter__(self): print("BEGIN") return self def __exit__(self, exc_type, exc_val, tb): if exc_type is None: print("COMMIT") else: print("ROLLBACK") return False

This pattern is used in SQLAlchemy, Django ORM, and most Python DB libraries. Now you know how it works inside.
