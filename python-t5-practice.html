✨ Solution

Solution
def __init__(self, name, tasks=None): self.name = name self.tasks = [] if tasks is None else tasks

Why: The default [] is created ONCE when the function is defined, not on every call. All instances using the default share that one list. Using None sidesteps the issue.

Design Transaction with validation + multiple constructors
🎯 Scenario: A payments service needs a clean Transaction class. Should support: validation (amount > 0), creating from a dict, creating from a UPI string like "PAY:1042:1499.50".
Task: Use @dataclass for storage, @staticmethod for validation, @classmethod for the two alternate constructors.
HintThree decorators in play. Validate inside the classmethods before returning cls(...).
🐍 transaction.py
loading runtime…
Output will appear here…

✨ Solution

Solution
@staticmethod def is_valid_amount(x): return isinstance(x, (int, float)) and x > 0 @classmethod def from_dict(cls, d): if not cls.is_valid_amount(d["amount"]): raise ValueError("Bad amount") return cls(d["id"], d["amount"]) @classmethod def from_upi_string(cls, s): _, id_str, amt_str = s.split(":") return cls.from_dict({"id": int(id_str), "amount": float(amt_str)})

Why: Static method for stateless validation (no self, no cls). Two classmethods for alternate constructors. from_upi_string reuses from_dict for validation.

🎯 Scenario: Common interview question. Design a Pipeline class that tracks retry attempts internally and refuses to retry beyond max_attempts. Must use a @property for can_retry.
Task: Class takes name and max_attempts. attempt() increments internal counter and returns True if allowed. can_retry is a property returning True/False. Test below should pass.
HintInternal counter self._attempts = 0. Property can_retry returns self._attempts < self.max_attempts.
🐍 retryable.py
loading runtime…
Output will appear here…

✨ Solution

Solution
class RetryablePipeline: def __init__(self, name, max_attempts): self.name = name self.max_attempts = max_attempts self._attempts = 0 @property def can_retry(self): return self._attempts < self.max_attempts def attempt(self): if not self.can_retry: return False self._attempts += 1 return True

Why: _attempts is internal (underscore convention). can_retry as property gives clean API. attempt() checks and increments. This pattern appears in real Airflow retry logic.
